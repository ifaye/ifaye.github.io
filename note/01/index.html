移动端1像素问题及解决方法

原因是移动设备分辨率的问题，二倍屏三倍屏，1px渲染出来的效果比实际粗。我常用的解决方案就是伪类 :before:after 和 transform ，优点就是兼容性好。


.scale-1px{
	 position: relative;
	 border:none;
 } 
.scale-1px:after{
	 content: '';
	 position: absolute; 
	 bottom: 0; 
	 background: #000; 
	 width: 100%; 
	 height: 1px;
}
2倍屏 @media only screen and (-webkit-min-device-pixel-ratio: 2.0) { 
	.scale-1px:after { -webkit-transform: scaleY(0.5); transform: scaleY(0.5); }
	}
3倍屏 @media only screen and (-webkit-min-device-pixel-ratio: 3.0) { 
	.scale-1px:after { -webkit-transform: scaleY(0.33); transform: scaleY(0.33); }
	}

@media 和 @import 媒体查询

					
//语法
@media mediatype and|not|only (media feature) {
	CSS-Code;
}
or
<link href="mystylesheet.css" rel="stylesheet" media="mediatype and|not|only (media feature)">
					

//示例
<link href="../style/m_index.css" rel="stylesheet" media="screen and (max-device-width:480px)">
//当设备是电脑屏幕、平板电脑、智能手机，且屏幕分辨率宽度不超过480px时引用改css

不固定大小的图片垂直居中左右居中

移动端可以用 flex 处理，PC 端则需要用到 vertical-align:middle


// &nbsp; 和 line-height:200px; 是重点				
#picbox{width:200px;height:200px; text-align:center; line-height:200px; border:1px #000 solid;}
#picbox img{vertical-align:middle;}

<div id="picbox">
  <img src="" alt="" />&nbsp;
</div>

固定高度内容居中

移动端可以用 flex 处理，PC 端则需要用到 display:table-cell


父元素 display:table，子元素 display:table-cell;vertical-align:middle;

	<div style="display:table-cell;vertical-align:middle; width:200px; height:200px;">

</div>
//注意 div 不能有 float 属性

火狐浏览器 表格 table td th 不支持 position

遇到一个火狐浏览器里的小 bug，在表格 table 里的 th 加了个相对定位，在里边的某个 div 里加了绝对定位，但是绝对定位不起作用，但在 ie 和谷歌浏览器里正常。解决办法：给 th 加了个 display 属性，让它为块状元素，即 display:block;
